## My process



Process



A brief overview of the steps taken:



Design Inspection: Reviewed Figma artboards to extract color palette, typography, spacing, and asset exports.



HTML Structure: Created semantic, accessible markup for the QR code card component.



Styling: Defined CSS custom properties for design tokens and used Flexbox to replicate the layout.



Responsiveness: Added media queries to adjust dimensions and font sizes for mobile and desktop.



Testing & Refinement: Validated pixel-perfect alignment against design previews and refined spacing.Built with

- Semantic HTML5 markup
- CSS custom properties (CSS variables)
- Flexbox for layout
- Media queries for responsiveness

### What I learned

- **Translating Figma designs**: Extracted exact colors, typography, spacing, and converted them into CSS variables.
- **Semantic HTML**: Structured content with meaningful elements for accessibility.
- **CSS variables**: Implemented a design system using custom properties for maintainable theming.
- **Responsive design**: Used Flexbox and media queries to adapt the card to mobile and desktop views.

### Continued development

- Enhance accessibility with ARIA labels and focus states.
- Add subtle hover/focus animations to improve interactivity.
- Explore CSS Grid for alternative layouts.
